How it works

From URL to evidence without unsafe actions

  1. Validate target. A11yway rejects private hosts, unsafe schemes, credentials, unsafe ports, and unsafe redirects.
  2. Run selected modules. The server uses the existing deterministic audit engine and records each module status.
  3. Review findings. AI Scout can summarize verified findings after the deterministic audit, but never replaces evidence.
  4. Download reports. Use generated HTML, Markdown, JSON, CSV, SARIF, and JUnit files when available.

What A11yway checks

Modules built for practical public-page review

Keyboard and rendered DOM

Focus traversal, loop detection, rendered HTML comparison, axe-core, and Chromium accessibility-tree evidence when browser mode is available.

Low vision and mobile

Contrast samples, visible focus indicators, zoom reflow, mobile viewport overflow, orientation review, and touch target checks.

HTML semantics

Forms, components, media, headings, labels, language changes, cognitive load signals, and Indic-language markup.

Evidence and reports

Findings can include selectors, snippets, WCAG criteria, browser observations, screenshots, focus overlays, and suggested remediation.

Safety and scope

Responsible public testing only

A11yway does not log in, submit forms, test payments, scan networks, bypass authentication, exploit weaknesses, or claim legal compliance certification.

  • Public pages only
  • No private systems
  • No form submissions
  • No security exploitation
  • Human review required
  • No legal certification claim
