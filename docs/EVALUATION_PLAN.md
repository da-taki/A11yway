# Evaluation

Use A11yway reports as a starting point for human review.

## Suggested flow

1. Run a focused audit on a public or permitted page.
2. Review strong findings first.
3. Check `needs_review` findings against the rendered page.
4. Group repeated findings by template or component.
5. Record whether each finding is confirmed, not reproducible, or outside scope.
6. Re-run after fixes and compare reports.

Do not convert repeatable heuristics into conformance claims. A repeated issue is easier to triage, but it still needs the right level of review.
